# Changelog - 2026-04-27

## [20:37] Hardened Docker image + Compose config

**类型**：infra
**概述**：Dockerfile 改为 non-root distroless 镜像并删除 CGO；Compose 补全必填环境变量和
healthchecks；CI 对齐 Go 1.26.1 并新增镜像构建 job；新增 Makefile targets。

**影响文件**：
- `.dockerignore`
- `.github/workflows/ci.yml`
- `Dockerfile`
- `Makefile`
- `docker-compose.yml`

**详情**：

Dockerfile:
- 两阶段构建：`golang:1.26.1-alpine` 构建层 + `gcr.io/distroless/static-debian12:nonroot` 运行层
- 禁用 CGO，`-ldflags "-s -w" -trimpath`，产物 strip 后约缩小 30%
- 运行用户改为非 root（distroless nonroot 用户）

docker-compose.yml:
- 新增必填 env 校验：`POSTGRES_PASSWORD`, `API_KEYS`, `CORS_ALLOWED_ORIGIN` 缺失会阻止启动
- 固定 `postgres:17.2-alpine`，加 healthcheck（pg_isready）、restart policy、命名 volume
- Ledger 服务 depends_on: postgres condition: service_healthy

CI:
- Go 版本对齐 `1.26.1`
- `sqlc diff` 步骤修正 working-directory 到 `postgres/`
- 新增 `image-build` job（仅主分支 / PR 触发）

Makefile:
- 新增 `build` / `test` / `vet` / `lint` / `sqlc` / `docker` targets；零依赖，`make help` 可列出

## [21:22] Cross-arch Dockerfile + multi-arch CI build

**类型**：fix / infra
**概述**：修复在 arm64 宿主机构建时二进制与运行时镜像架构不匹配的问题；CI 改为同时产出 `linux/amd64` + `linux/arm64` 两架构镜像。

**影响文件**：
- `.github/workflows/ci.yml`
- `Dockerfile`

**详情**：
- Dockerfile 用 `ARG TARGETOS / TARGETARCH` 代替硬编码 `amd64`；`BUILDPLATFORM` 固定在构建层以支持原生交叉编译（不经 QEMU）
- CI job 加入 `docker/setup-qemu-action` 和 `docker/setup-buildx-action`，`platforms: linux/amd64,linux/arm64`

Verification: `docker buildx build --platform linux/amd64,linux/arm64 --load` ✅
