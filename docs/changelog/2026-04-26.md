**详情**：

Backend audit fixes:
- `Reserve` now takes a per-(holder, currency) advisory lock and re-checks `available = totalBalance - SUM(active reservations)` inside the lock to close TOCTOU.
- `LedgerStore.ExecuteTemplateBatch` posts multi-step deposit-tolerance plans atomically; `postJournalWithQueries` shared between single and batch paths.
- `GetBalance` runs checkpoint + entry-sum queries inside a `REPEATABLE READ` transaction.
- Webhook ingress capped at 1MB via `io.LimitReader`; CORS warns when `CORS_ALLOWED_ORIGIN` is unset.
- `convert.go` surfaces JSON unmarshal / numeric conversion errors via `slog`; flags `float64` precision-loss path.
- Webhook delivery uses `retryDelay(evt.Attempts)` for true exponential backoff.

Frontend DRY refactor:
- New viem-based `web/src/lib/utils/` package with decimal arithmetic, display formatting, validation, clipboard, date, cn, address helpers.
- `useLedgerMutation` wrapper centralises cache invalidation across 14 mutations (deposits, withdrawals, journals, reservations).
- Shared `<ErrorState>`, `<EmptyState>`, and `alert-dialog` components applied to 11 page components, eliminating ~150 lines of duplicated state JSX.
- Bumped `tsconfig.json` target to `ES2020` for native BigInt.
