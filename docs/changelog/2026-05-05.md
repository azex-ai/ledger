**详情**：
- **幂等三态语义**：同 key + 同 payload → 重放成功；同 key + 异 payload → `core.ErrConflict`；新 key → 新建。比对走 `journalEntryFingerprint`（字符串化 decimal + 排序）+ metadata canonical JSON 比对
- **幂等 Advisory Lock**：`pg_advisory_xact_lock(hashtextextended(key, 0))` 串行化同 key 并发，避免两请求都过初查、都进业务逻辑、最后一方在 unique violation 上回滚浪费工作的场景
- **Event/Journal/Booking 原子链接**：`PostJournal` 携 `EventID` 时同 tx 内回填 `events.journal_id` + `bookings.journal_id`；SQL 层 `WHERE journal_id IS NULL` 乐观并发；`bookings.journal_id` set-once（每个 booking 生命周期最多一次 journal-bearing transition）
- **`account_holder = 0` 拒绝**：core 校验 + 5 张表 CHECK 约束（journal_entries / reservations / deposits / withdrawals / bookings）；迁移 `NOT VALID` 立即生效新写入、`VALIDATE` 单独验证历史数据，避免长时间锁表
- **测试**：`go vet ./...` 干净，`go test -race -count=1` 全部通过（core/presets/channel/onchain unit + postgres 集成 86s + server + service）
